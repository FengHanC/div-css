order-demo — 接口幂等性最佳实践

PHP 8 ThinkPHP 8 MySQL

项目背景

在电商系统中，订单接口的幂等性是核心难题之一。用户重复提交、网络重试等都可能导致重复下单。这个项目展示了如何通过 request_id + 数据库唯一索引双重保障来解决这个问题。

技术亮点

  • 幂等键机制 — 客户端传入 request_id，服务端去重
  • 唯一索引兜底 — 数据库层面防止并发穿透
  • 库存扣减事务 — 并发冲突时自动回滚，保证数据一致性

接口示例

POST /order/create
{
  "user_id": 1,
  "product_id": 100,
  "request_id": "uuid-xxx"
}

// 响应
{
  "code": 0,
  "msg": "下单成功",
  "order_no": "20240418120000123456"
}
